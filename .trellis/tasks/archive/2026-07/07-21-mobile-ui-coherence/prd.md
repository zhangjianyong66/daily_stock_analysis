# 修复移动端首页 UI 割裂问题

## Goal

将移动端首页恢复到提交 `2cd093e` 之前的原版 UI 与交互结构，并在不改变信息架构的前提下，只对组件宽高、间距和移动触控尺寸做克制的小幅优化，消除最近一次大改造成的割裂感。

## Background and Confirmed Facts

- 问题出现在提交 `2cd093e feat(web): 优化移动端首页 UI 与交互` 之后，改动集中在 Web 首页、移动历史条、报告摘要、抽屉和底部报告操作栏。
- 当前移动端首页同时存在四个专用交互层：顶部搜索与“更多操作”、独立的横向“快速访问”带、报告内“查看更多详情”折叠入口、固定底部图标操作栏。
- “快速访问”使用上下边框和独立背景形成整宽分区；报告详情入口使用带边框、背景和阴影的独立卡片；固定底栏再次使用边框、背景、阴影和模糊效果。多个强边界连续出现，是页面产生分段感的明确结构风险。
- 报告策略、资讯、上下文、诊断和追溯在移动端默认全部隐藏在“查看更多详情”之后，首屏内容与完整报告之间形成额外的信息断点。
- 上一任务以功能可达、触控尺寸、无横向溢出和桌面兼容为主要验收目标；现有测试证明功能契约成立，但没有定义或验证页面整体的视觉连续性。
- 用户已明确选择恢复原版 UI，不继续围绕现有移动专用分层做二次设计；允许的小优化限于尺寸、间距等局部样式。
- 用户确认保留 `2cd093e` 为通用 `Drawer` 增加的焦点约束、Escape 关闭和焦点恢复能力，但撤销仅服务于移动“分析选项”的底部 Drawer 接口与样式。
- 用户确认 320px 窄屏继续显示完整操作文字，通过自然换行和间距控制密度，不使用图标替代文字。

## Requirements

- R1 以 `2cd093e^` 的 `HomePage` 和 `ReportSummary` UI/交互为恢复基线，不重新设计移动端信息架构。
- R2 移除本次大改新增的移动专用横向快速访问带、更多操作底部抽屉、报告详情总折叠入口和固定底部图标操作栏，恢复原版入口位置与内容顺序。
- R3 保留股票搜索、单股分析、历史报告访问、策略/通知/批量/市场复盘、报告深入阅读和报告操作等原有能力。
- R4 小优化允许调整组件宽高、内外间距、窄屏换行/滚动、必要的触控尺寸，以及为解决 320px 布局问题所需的轻微 DOM 排列；不得改变入口位置、操作层级或内容顺序，也不得新增页面层级、抽屉、折叠区、悬浮栏或新的操作路径。
- R5 继续满足 320px 兼容下限、无页面级横向溢出和桌面布局不回归；触控尺寸优化不能以改变原版信息架构为代价。
- R6 最终方案必须包含真实移动视口的页面截图或等价可视验收证据，不能只依赖 DOM 测试和构建通过。
- R7 通用 Drawer 继续满足打开后聚焦、Tab/Shift+Tab 约束、Escape 关闭和关闭后恢复触发器焦点；不再支持本任务引入的 `side="bottom"` 分支。
- R8 保留 `ReportOverview` 中响应式字号、间距、长股票名称换行等符合“小优化”边界的改动；报告内容层级和出现顺序恢复原版。

## Acceptance Criteria

- [x] 375×812 和 390×844 视口下的首页恢复为 `2cd093e^` 的原版结构和操作位置，只存在可明确列举的尺寸/间距微调。
- [x] 页面不再显示横向“快速访问”带、“分析选项”底部抽屉、“查看更多详情”总折叠入口或固定底部图标操作栏。
- [x] 320px 下顶部与报告操作区保留完整文字；允许自然换行，但不得使用仅图标按钮替代原版命令。
- [x] 所有现有首页能力仍然可达，且桌面端结构和行为保持兼容。
- [x] 320px 宽无页面级横向溢出，控件文本不遮挡、不截断关键命令。
- [x] 受影响的 HomePage、历史条、报告摘要和 Drawer 测试通过，Web lint/build 通过。
- [x] 已检查 320×720、375×812、390×844 和 1280×900 视口截图；临时验收图按仓库规则未写入版本目录。

## Out of Scope

- 不改变后端 API、分析任务、报告数据结构或权限契约。
- 不借此重做全站主题、桌面端信息架构或其他页面。
- 不新增更多卡片、抽屉、折叠区或悬浮层。
- 不继续迭代本次新增的移动端信息架构。
