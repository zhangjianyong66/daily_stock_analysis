# 技术设计：恢复移动端首页原版 UI

## 1. 恢复基线与边界

- 以 `2cd093e^` 中的 `HomePage`、`ReportSummary`、`StockBarItem` 和历史侧栏呈现为原版基线，选择性撤销 `2cd093e` 的移动专用信息架构。
- 不直接执行整笔 Git revert：保留用户确认的通用 Drawer 焦点管理，以及 `ReportOverview` 的响应式字号、间距和长名称换行优化。
- 不改变任何 API、store、分析任务、报告数据或路由契约。

## 2. 组件调整

### HomePage

- 移除 `MobileStockStrip`、移动“更多操作”按钮/状态/底部 Drawer，以及固定底部报告操作栏。
- 恢复原版顶部结构：历史入口、股票搜索、策略、通知、批量分析、大盘复盘、单股分析均直接可见。
- 恢复原版移动历史侧栏视觉；从历史侧栏进入运行流时继续先关闭历史层，避免两个叠层并存。
- 恢复报告上方带文字的操作按钮组，移动端允许自然换行；不使用仅图标按钮。
- 微调移动端高度、间距、最小宽度和换行规则，保证 320px 不横向溢出；桌面断点保持原布局。

### ReportSummary / ReportOverview

- `ReportSummary` 恢复所有报告模块顺序显示，移除“展开/收起深入分析”总开关。
- 保留 `ReportOverview` 已有的移动字号、间距、长股票名称换行和窄屏栅格间距优化，因为它们不改变信息结构。

### History / Drawer

- 删除只服务于横向快速访问带的 `MobileStockStrip`，撤销 `StockBarItem` 的 `compact` 变体和相关导出。
- `Drawer` 保留焦点进入、焦点循环、Escape 关闭、关闭后焦点恢复和 `dialogId`/`aria-controls` 关联能力；仅移除 `side="bottom"` 与底部视觉分支。
- 调整 Drawer 测试，使其覆盖保留的通用可访问性行为，而不依赖底部样式。

## 3. 文案、测试与文档

- 删除只服务于被撤销结构的中英文文案键。
- HomePage 测试恢复原版结构断言，并新增/保留“移动控件文字可见、工具栏可换行、无移动专用分层”的回归断言。
- `docs/CHANGELOG.md` 将未发布的“新增移动专用结构”条目改为恢复原版并优化窄屏尺寸/间距的准确描述。

## 4. 风险与回滚

- 主要风险：选择性恢复遗漏移动状态或文案；320px 工具栏换行后高度失控；测试仍绑定已删除结构；通用 Drawer 可访问性被误删。
- 通过源码搜索确保 `MobileStockStrip`、底部 Drawer、详情总折叠和固定操作栏无残留，通过定向测试和真实视口截图验证。
- 本任务不涉及数据迁移。需要回滚时，可按文件恢复本任务改动，不影响后端和用户数据。
