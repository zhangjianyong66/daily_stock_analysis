# 移动端首页 UI 与交互优化

## Goal

优化 DSA Web 首页在手机上的信息层级和操作路径，让用户可以快速查看自选/历史个股状态并启动单只股票分析，同时保持桌面端现有信息架构和行为稳定。

## Background and Confirmed Facts

- 首页实现位于 `apps/dsa-web/src/pages/HomePage.tsx`，当前同时承载股票搜索、策略选择、通知开关、批量分析、市场复盘、任务流、历史个股栏和报告摘要。
- `Shell` 在窄屏提供全局侧滑导航；首页自身还提供历史个股栏抽屉，移动端需要避免两套抽屉造成层级混乱。
- 报告摘要由 `ReportSummary` 渲染，全文报告和运行流分别使用抽屉，现有 API/数据模型可以继续复用。
- 当前任务只针对移动端体验；桌面端结构和交互不作为重做目标。

## Requirements

- R1 首屏主任务：移动端首页优先呈现自选/历史个股状态、股票搜索/选择和单只分析入口。
- R2 操作分层：首屏保留搜索和主分析；策略、通知、批量分析、市场复盘进入“更多操作”底部抽屉或等价移动菜单，不能丢失原能力。
- R3 个股快速访问：移动端个股栏以横向可滑动、可触控的紧凑条目呈现；点击条目可查看对应报告，并保留启动分析/进入详情的路径。
- R4 报告阅读：移动端默认显示紧凑摘要，优先展示评分、趋势、操作建议；重分析、问 AI、历史趋势、全文等动作收纳到固定底部操作栏或更多菜单，全文报告继续以全屏抽屉打开。
- R5 触控与安全区：主要触控目标不小于 44px；底部操作栏适配安全区，不遮挡报告内容；软键盘弹出时搜索和主分析仍可操作。
- R6 响应式范围：以 375×812 和 390×844 为主验收尺寸，320px 宽为兼容下限，不得出现横向页面溢出；平板和桌面沿用现有布局。
- R7 可访问性与兼容：保留现有 aria 标签、键盘导航和中英文文案机制；不新增后端接口，不改变已有 API 字段和桌面行为。

## Acceptance Criteria

- [x] 在 375×812、390×844 和 320px 宽视口下，首页无页面级横向溢出，首屏搜索框和主分析按钮可见且可操作。
- [x] 移动端首屏不同时展开策略、通知、批量分析、市场复盘等次要控件；通过“更多操作”仍可完成原有操作并显示加载、成功、失败状态。
- [x] 个股条目支持横向滑动，触控区域不小于 44px，点击后能打开对应历史报告或进入现有分析路径。
- [x] 有报告时，移动端先呈现紧凑摘要和核心结论；底部操作栏不会覆盖最后一段内容，四类报告操作仍可达成。
- [x] 全文报告、运行流、确认弹窗和错误提示在移动端保持可关闭、可滚动且不被底部操作栏遮挡。
- [x] 桌面视口回归现有首页布局；相关 HomePage、报告和交互测试通过，Web lint/build 通过。
- [x] 不引入新的 API、持久化字段或桌面端专属行为变化。

## Out of Scope

- 不重做桌面端导航、桌面端报告信息架构或全站视觉主题。
- 不新增实时行情、推送、离线缓存或后端能力。
- 不改变分析任务、报告结构、权限和 API 契约。

实现阶段如发现现有组件无法满足安全区或抽屉语义，只允许在 Web 组件边界内补充兼容样式/交互，不扩展到 API 或桌面端重构。
