# 个股栏排序功能

## Goal

在 Web 首页个股栏中提供可理解、可切换且能记住用户选择的排序方式，帮助用户按时间、分析活跃度、最新情绪分或名称快速浏览当前可见的分析对象。

## Background

- 个股栏由 `apps/dsa-web/src/components/history/StockBar.tsx` 渲染；首页会把最新大盘复盘作为 `MARKET` 条目合并进列表。
- 当前默认加载最近 90 天内出现过的最多 200 个去重股票条目，后端 `GET /api/v1/history/stocks` 固定按最近分析时间降序返回，不支持排序参数。
- `StockBarItem` 已提供本功能所需的 `lastAnalysisTime`、`analysisCount`、`sentimentScore`、`stockName` 和 `stockCode`，本次不需要新增数据字段或扩大数据范围。
- 个股栏已有代码/名称过滤、全选、单条/批量删除和点击查看报告交互，排序必须与这些行为共存。

## Requirements

### R1 排序方案

- 提供以下五种方案：
  - `最近分析`：最近分析时间从新到旧，作为无有效持久化偏好时的默认值。
  - `最早分析`：最近分析时间从旧到新。
  - `分析次数最多`：历史分析次数从高到低。
  - `情绪分最高`：最新情绪评分从高到低。
  - `名称/代码`：优先按股票名称升序，名称缺失时使用股票代码。
- 排序只作用于当前已加载的最近 90 天、最多 200 个条目，不改变 API、数据库查询、分页或历史可见范围。
- 最新大盘复盘 `MARKET` 参与全部排序方案，不固定置顶或置底。

### R2 稳定顺序与边界值

- 当前排序字段缺失、时间无效或值不可用时，该条目始终排在字段有效的条目之后。
- 除“名称/代码”外，主要排序字段相同时先按最近分析时间从新到旧，再按股票代码升序，保证确定性顺序。
- “名称/代码”中名称缺失时以股票代码参与排序；名称相同时直接按股票代码升序。
- 数值 `0` 是有效值，不按缺失处理。
- 名称排序跟随当前界面语言使用浏览器本地化自然比较，并启用数字自然顺序；不额外维护拼音表。

### R3 交互与状态

- 在个股栏搜索框下方提供紧凑下拉选择器，明确显示“排序”标签和当前方案；桌面端与移动端采用同一交互。
- 排序控件支持键盘操作和读屏访问，中文和英文界面均有完整标签与选项文案。
- 用户切换方案后，当前列表立即在前端重排，不重新请求数据。
- 排序选择写入当前浏览器的 `localStorage`；没有记录、记录无效或存储不可用时安全回退到“最近分析”。不新增账号级或后端配置。
- 自动刷新或新分析完成后保留当前排序选择，并基于更新后的条目重新计算顺序。
- 排序不得改变条目身份、当前报告选中状态、过滤结果、选择集合、删除目标或点击查看报告行为。

### R4 文档与验证

- 在 `docs/CHANGELOG.md` 的 `[Unreleased]` 扁平列表中记录这项用户可见改进。
- 前端测试覆盖默认排序、五种方案、缺失/无效/相同值、持久化回读、刷新后保留选择，以及中英文控件文案。
- 最终执行受影响前端测试，并按 Web 改动门禁执行 lint 和 build。

## Acceptance Criteria

- [x] 首次使用时，普通个股与 `MARKET` 按最近分析时间从新到旧排列，下拉框显示“最近分析”。
- [x] 五个排序选项都能立即重排当前条目，且不会发起新的 stock-bar API 请求。
- [x] 切换方案后刷新页面、自动刷新数据或完成新分析，仍沿用已保存的有效排序方案。
- [x] 无效持久化值、存储访问失败、缺失字段、非法时间、并列值和数值 `0` 均按 R2/R3 的规则处理，列表不崩溃且顺序确定。
- [x] “名称/代码”在中英文界面按对应语言进行自然排序，名称缺失时按代码排序。
- [x] 过滤、当前报告选中、全选、单条/批量删除和点击查看报告在任一排序方案下保持正确。
- [x] 排序控件在桌面端和移动端可用，并具有可识别标签、当前值、键盘操作能力和中英文文案。
- [x] `docs/CHANGELOG.md` 已更新；定向测试、Web lint 和 Web build 通过。

## Out Of Scope

- 服务端排序参数、数据库查询排序、全历史股票排序或新的分页机制。
- 账号级偏好同步、多设备同步或设置页配置。
- 额外排序方案、独立升降序切换按钮、拖拽自定义顺序。
- 改变个股栏最近 90 天和最多 200 个条目的现有加载范围。
