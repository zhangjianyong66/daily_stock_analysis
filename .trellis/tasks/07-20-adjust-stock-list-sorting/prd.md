# 调整个股栏排序选项

## Goal

精简 Web 首页个股栏的排序方案，使用户可以按最近分析、情绪分高低或名称/代码组织列表，同时移除价值较低的历史时间正序和分析次数排序。

## Background

- 当前默认排序为“最近分析”，排序偏好通过 `dsa.stockBarSort.v1` 保存在浏览器 `localStorage`。
- 当前提供“最近分析”“最早分析”“分析次数最多”“情绪分最高”“名称/代码”五个选项。
- 用户明确要求删除“最早分析”“分析次数最多”，并新增“情绪分最低”。

## Requirements

- 默认排序继续为“最近分析”。
- 排序下拉框按“最近分析”“情绪分最高”“情绪分最低”“名称/代码”的顺序展示四个选项。
- “情绪分最低”按有效情绪分升序排列。
- 保留现有排序偏好持久化、自动刷新后重排、中英文界面和 MARKET 项统一排序行为。
- 删除的排序值不得继续作为可选或可执行排序方案。
- 浏览器中已保存的 `oldest` 或 `most-analyzed` 视为不受支持的旧值，并统一回退到默认“最近分析”。
- 缺失或非法情绪分继续排在所有有效情绪分之后，`0` 仍作为有效值。
- 情绪分并列时继续按最近分析时间、再按代码确定稳定顺序。
- 不调整 API、数据库或首页历史数据加载范围。

## Acceptance Criteria

- [x] 排序下拉框在中英文界面均只显示四个目标选项，且“最近分析”为默认值。
- [x] 选择“情绪分最低”后，有效情绪分从低到高排列，`0` 正确参与排序，缺失或非法值位于末尾。
- [x] “情绪分最高”“最近分析”“名称/代码”的既有行为保持不变。
- [x] 新排序选择可持久化，并在数据刷新或组件重新挂载后继续生效。
- [x] 浏览器中已保存的 `oldest` 或 `most-analyzed` 在升级后回退为“最近分析”。
- [x] 定向测试、Web lint 与 Web build 通过，`docs/CHANGELOG.md` 同步反映最终选项。

## Out Of Scope

- 不新增服务端排序或跨设备同步。
- 不改变排序控件布局和个股栏其他交互。
