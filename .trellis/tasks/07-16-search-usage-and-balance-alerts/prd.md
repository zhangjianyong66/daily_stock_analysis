# 增加搜索调用用量记录与余额告警

## Goal

为 Anspire 等搜索供应商建立可核查的本地调用账本，按每次真实外部请求记录调用次数和脱敏后的完整业务出入参，帮助用户独立复核未提供调用次数查询的供应商是否存在异常额度消耗；同时对余额不足、认证失效等持续性故障提供主动提醒，避免分析流程静默退化为“0 条搜索结果”。

## Background

- 当前 Web“用量”页面只展示 `llm_usage` 的 LLM Token 汇总与最近调用记录。
- 搜索调用主要存在于文本日志和单次分析诊断中，没有可分页、筛选、导出的供应商调用账本。
- 2026-07-16 Anspire Search 返回“可用免费额度 0.00，充值余额 0.00”，但现有代码按 HTTP 401 显示为“API KEY 无效”；同一分析任务的五个搜索维度全部失败后，用户只看到“共 0 条结果”。
- Anspire 未提供可供用户核对的搜索调用次数查询，而余额消耗较快；用户需要本地真实出站请求记录核查调用次数和供应商响应。
- Anspire 当前只用于 Search，LLM 使用其他渠道，因此搜索调用量与 LLM Token 用量必须保持独立统计语义。

## Requirements

### R1. 计数口径与覆盖范围

- 每次真实发往搜索供应商的外部请求独立记一条账，包括成功、失败、自动重试、备用 Key、供应商 fallback 和多实例尝试。
- 缓存命中、本地结果过滤、数据库读取、文章正文补抓和 SearXNG 公共实例目录刷新不计入供应商搜索调用次数。
- 覆盖股票分析、Agent 搜索、行情增强兜底及其他通过项目搜索供应商发出的真实搜索请求。
- 每条记录标注调用来源；页面分别展示“真实外部请求数”和“业务搜索任务数”。
- 项目控制的底层 HTTP 重试必须逐次记录。当前 Tavily、SerpAPI SDK 可观察的单次搜索 HTTP 请求按一次记录。

### R2. 审计字段与出入参快照

- 每条物理请求记录：毫秒级时间、供应商、接口、HTTP 方法、业务搜索任务 ID、逻辑请求 ID、物理请求序号、重试/回退序号、调用来源、股票代码、股票名称、搜索维度、时间窗口、API Key HMAC 指纹和查询 HMAC。
- 每条记录同时包含 HTTP 状态、供应商业务错误码、成功状态、耗时、结果数量、错误类别、安全化错误摘要，以及供应商返回的 Request ID/Trace ID（如果存在）。
- 保存经深度脱敏的完整业务入参：接口地址、完整搜索词、查询参数、允许保留的请求头和请求体。
- 保存经深度脱敏的完整业务出参：HTTP 状态、允许保留的响应头、完整 JSON 业务响应和搜索结果；非 JSON 响应保存脱敏文本。
- 脱敏必须递归覆盖嵌套对象、列表、URL 查询参数和响应头，删除或替换 Authorization、API Key、Cookie、Token、签名、Webhook 和其他凭据。无法确认安全的敏感字段默认不落库。
- 脱敏后的完整请求/响应 JSON 以明文保存，不做应用层加密。用户已接受数据库文件持有者能够读取查询内容和搜索结果的风险。
- 请求快照上限为 256 KiB，响应快照上限为 2 MiB。超限时保存脱敏后的前部内容，并记录完整脱敏快照的原始字节数、截断标记和 SHA-256。

### R3. 持久化可靠性与保留策略

- 每次供应商请求完成后同步写入审计记录，再向上层返回搜索结果；不得以进程内异步队列作为唯一持久化机制。
- 审计写入保持 fail-open：写入失败不得改变搜索结果或中断分析任务，但必须记录错误、累计可见的丢失记录计数并触发审计链路故障提示。
- 搜索调用账本默认永久保留，不执行自动过期或后台清理。
- 不从现有文本日志回填历史数据。页面明确显示正式审计记录的起始时间。

### R4. 错误分类与故障生命周期

- 错误类别至少区分：`quota_exhausted`、`auth_invalid`、`permission_denied`、`account_disabled`、`rate_limited`、`timeout`、`connection_error`、`provider_5xx`、`invalid_response` 和 `other`。
- 分类优先使用供应商业务响应语义，再使用 HTTP 状态；Anspire HTTP 401 响应中明确出现余额/额度耗尽时必须归类为 `quota_exhausted`，不能归类为 `auth_invalid`。
- 故障按“供应商 + Key HMAC 指纹 + 错误类别”跟踪。
- 余额不足、额度耗尽、API Key 无效、权限不足和账户停用首次发生时立即触发故障。
- 限流、超时、连接失败和供应商 5xx 在同一供应商与 Key 指纹 10 分钟内连续 3 次后触发故障。
- 正常返回 0 条结果或结果被本地时效/相关性规则过滤为空只记录调用结果，不触发故障。
- 同一 Key 后续成功可解除相应故障；Key 从当前配置中移除时也视为恢复。
- 供应商总体状态聚合为“正常 / 降级 / 不可用”：部分 Key 故障为降级，所有已配置 Key 均不可用为不可用。

### R5. 三层提醒与通知降噪

- 用量分析页面顶部展示未恢复故障卡片。
- Web Shell 全局展示未恢复故障警告；用户可在当前浏览器会话关闭，但后续会话仍会再次感知持续故障。
- Web 在启动、页面重新获得焦点和每 60 秒轮询一次故障状态；用量页面手动刷新时立即更新。
- 已配置通知渠道时，立即发送持续性故障通知；同一“供应商 + Key 指纹 + 错误类别”24 小时最多发送一次。
- 故障恢复后发送一次恢复通知。
- 通知内容只包含供应商、Key 短指纹、错误类别、时间和用量页面入口，不包含完整查询、请求/响应快照或凭据。

### R6. 用量分析页面与导出

- 将页面从“Token 用量监控”升级为“用量分析”，使用“LLM 用量”和“搜索调用”两个独立 Tab；现有 LLM Token 能力保持兼容。
- 搜索 Tab 展示：审计起始时间、审计链路健康状态、真实外部请求数、业务搜索任务数、成功数、失败数、成功率、供应商分布、Key 指纹分布、调用来源分布和最近调用明细。
- 支持今日、最近 7 天、本月、全部和自定义北京时间起止日期时间。
- 支持按供应商、调用来源、成功状态、错误类别和 Key 指纹筛选；明细使用服务端分页。
- 主表仅展示核账摘要。点击记录打开详情抽屉，分别查看和复制请求/响应 JSON；截断快照显示原始大小和 SHA-256。
- CSV 按当前筛选条件导出全部核账摘要字段，不嵌入完整请求/响应快照。
- 每条记录可下载包含审计元数据和脱敏完整出入参的单条 JSON 文件。

### R7. 权限与安全边界

- 搜索汇总、调用次数和脱敏故障摘要沿用现有用量页面的可选认证边界。
- 完整请求/响应详情、复制、CSV 导出和单条 JSON 下载必须满足：`ADMIN_AUTH_ENABLED=true` 且当前请求携带有效管理员会话。没有桌面端例外。
- 未启用管理员认证或当前未登录时，Web 禁用敏感详情与导出操作，并提供可操作提示；服务端必须独立执行权限校验，不能只依赖前端禁用。
- 原始 API Key、认证头、Cookie、Token、签名和其他凭据不得出现在数据库快照、公开 API、CSV、JSON 下载、通知或普通日志中。

### R8. 兼容、文档与验证

- 新表通过现有 SQLAlchemy `create_all` 兼容旧数据库；不得要求用户手工迁移或清空数据库。
- 审计记录或提醒链路的失败不得改变现有搜索 fallback、结果过滤和分析流程语义。
- 同步更新中英文 Web 文案、`docs/full-guide.md`、`docs/full-guide_EN.md`、API 文档说明和 `docs/CHANGELOG.md`。
- UI 变更的 PR 描述需要附用量分析页面和全局故障提示截图；截图不提交仓库。

## Acceptance Criteria

- [ ] AC1：一次业务搜索产生多维请求、自动重试或供应商 fallback 时，每个真实外部请求均有独立记录，缓存命中和本地处理不计数。
- [ ] AC2：Anspire HTTP 401 且响应正文为余额不足时，记录分类为 `quota_exhausted`，UI/通知显示“余额不足/额度耗尽”，不显示为“API KEY 无效”。
- [ ] AC3：每条记录能够重建业务任务、逻辑请求、物理尝试、供应商、Key 指纹、请求时间、状态、耗时、结果数和错误原因。
- [ ] AC4：正常大小的脱敏请求/响应完整保存；超限快照显示截断状态、原始大小和 SHA-256，且不阻断搜索。
- [ ] AC5：凭据泄漏回归测试覆盖嵌套 JSON、列表、URL 参数、请求头、响应头和供应商错误文本；数据库/API/导出/日志均不包含测试凭据。
- [ ] AC6：审计写入成功时在搜索结果返回上层前完成；模拟 SQLite 写入失败时原搜索结果不变，同时日志和页面能看到审计缺口。
- [ ] AC7：余额、认证、权限类错误首次发生即产生故障；瞬时错误只有 10 分钟内连续 3 次后产生故障；空结果不误报。
- [ ] AC8：多 Key 场景能分别查看调用量和故障；部分 Key 失效显示供应商降级，全部 Key 失效显示不可用；成功或移除 Key 后恢复。
- [ ] AC9：同一故障 24 小时内最多发送一次外部通知，恢复时发送一次恢复通知；Web 全局提示最迟约 60 秒出现。
- [ ] AC10：用量分析双 Tab 保留现有 LLM 用量功能，并提供搜索汇总、筛选、服务端分页、详情抽屉、CSV 和单条 JSON 下载。
- [ ] AC11：搜索调用支持今日、最近 7 天、本月、全部及自定义北京时间范围；CSV 与页面筛选结果一致。
- [ ] AC12：未启用管理员认证或未登录时，敏感详情及导出 API 返回 401/403；管理员登录后可访问。前端禁用状态与服务端一致。
- [ ] AC13：旧数据库启动后自动创建新表；不回填旧日志，页面展示首条审计记录时间。
- [ ] AC14：后端定向测试、完整后端门禁、Web 定向测试、lint 和 build 通过；未执行的在线供应商验证有明确说明。

## Out of Scope

- 自动充值或调用供应商账单/余额查询 API。
- 在供应商未返回稳定计费字段时估算货币成本。
- 从历史文本日志回填正式审计记录。
- 整表防篡改哈希链、每日签名清单或外部时间戳存证；快照 SHA-256 只用于标识超限前的完整脱敏内容。
