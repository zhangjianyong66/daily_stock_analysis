# 首页个股栏置顶

## Goal

允许用户在首页“个股”栏中将常看的 ETF 或股票固定到列表顶部，避免它们因分析时间、情绪分或名称排序变化而难以查找。

## Background

- 首页个股栏展示最近 90 天内有历史分析记录的标的，并提供“最近分析 / 情绪分最高 / 情绪分最低 / 名称代码”四种浏览器本地排序偏好。
- ETF 与普通股票使用同一个列表，当前列表接口不提供可靠的资产类型字段。
- 首页还会合并一个代码为 `MARKET` 的大盘复盘项，它不是可置顶的真实标的。
- 置顶是当前浏览器的界面偏好，不需要跨浏览器、跨设备或登录会话同步。

## Requirements

- 置顶能力适用于个股栏中的所有真实标的，ETF 是主要使用场景，不要求识别或限制资产类型。
- 每个 `stockCode !== 'MARKET'` 的标的卡片始终显示可操作的图钉按钮；按钮状态明确区分置顶与未置顶，并提供中英文提示及无障碍名称。
- 点击图钉立即切换置顶状态，不打开该标的的历史详情，也不需要确认弹窗。
- 所有置顶标的排列在所有未置顶标的之前；不增加“置顶 / 其他”分组标题或分隔区。
- 置顶组和未置顶组内部均使用当前选择的既有排序规则；不记录独立置顶顺序，不提供拖拽排序。
- 搜索过滤先于置顶分组：只有代码或名称匹配搜索条件的标的参与排序，未匹配的置顶标的不额外显示。
- `MARKET` 不显示置顶按钮、不写入置顶集合，并继续作为未置顶项参与既有排序。
- 置顶状态保存在当前浏览器；刷新页面或重启浏览器后继续生效，清理站点数据或更换浏览器/设备后不要求保留。
- 浏览器存储不可用或已有数据损坏时，界面应安全退化为空置顶集合；写入失败时当前页面内的置顶交互仍然有效。
- 置顶数量不设上限。合法但当前不可见的标的代码继续保留，历史删除或最近 90 天窗口变化不隐式清除其置顶偏好，标的再次出现时自动恢复置顶。

## Acceptance Criteria

- [x] 用户可通过每行始终可见的图钉按钮置顶和取消置顶真实标的，按钮具有中英文提示及可访问名称，且操作不触发历史详情打开。
- [x] 置顶后标的立即移动到未置顶标的之前；取消置顶后立即回到当前排序规则决定的位置。
- [x] 多个置顶标的在四种排序选项下始终先于未置顶标的，且两个分组内部都遵循当前排序的既有并列与缺失值规则。
- [x] 搜索只显示匹配项，匹配结果内仍保持置顶优先；列表刷新后置顶分组继续生效。
- [x] `MARKET` 不显示置顶按钮，不能被损坏的本地数据置顶，并保持既有排序行为。
- [x] 页面重新挂载或刷新后从浏览器恢复置顶集合；非法 JSON、非数组、非字符串、空代码、重复代码和 `MARKET` 值均被安全忽略。
- [x] 浏览器存储读写抛错不会导致组件渲染或点击操作报错。
- [x] 现有删除、批量选择、历史打开、四种排序和中英文界面行为保持兼容。

## Out of Scope

- 服务端持久化、跨设备同步或按登录用户保存偏好。
- 仅限 ETF 的资产识别、API / Schema / 数据库改动。
- 手动拖拽排序、置顶顺序管理、置顶数量上限和批量置顶。
- 独立置顶分组标题、分隔线或跨页面置顶入口。
